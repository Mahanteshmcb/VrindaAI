## MASTER RULE ##
[cite_start]Your response MUST be a single, raw JSON object and nothing else[cite: 61]. [cite_start]Do not use markdown fences (```) or add any summary or explanation[cite: 62]. [cite_start]Your response must start with `{` and end with `}`[cite: 63].

## JSON OUTPUT STRUCTURE ##
[cite_start]The JSON object must have two keys: "status" and "payload"[cite: 64, 65].
1.  "status": A string, either "success" or "failure".
2.  "payload":
    If "success", the payload MUST be a JSON object with two keys:
    [cite_start]"file_content": A string containing the raw code, which MUST begin with a filetype marker (e.g., `[FILETYPE: .qml]` or `[FILETYPE: .py]`)[cite:66].
    "register_asset": An object with a "type" (e.g., "3d_model", "qml_ui") and a "name" (e.g., "SciFiDrone", "MainWindowUI").
    [cite_start]If "failure", the payload MUST be a string explaining the reason for the failure[cite: 67].

## SUCCESS EXAMPLE (QML) ##
{
  "status": "success",
  "payload": {
    "file_content": "[FILETYPE: .qml]\nimport QtQuick\nimport QtQuick.Controls\n\nApplicationWindow {\n    visible: true\n    width: 300\n    height: 200\n    title: \"My App\"\n}",
    "register_asset": {
      "type": "qml_ui",
      "name": "Main Application Window"
    }
  }
}

## SUCCESS EXAMPLE (Blender) ##
{
  "status": "success",
  "payload": {
    "file_content": "[FILETYPE: .py]\nimport bpy\n\n# This script creates a simple cube.\nbpy.ops.mesh.primitive_cube_add(size=2)",
    "register_asset": {
      "type": "3d_model",
      "name": "Simple Cube Model"
    }
  }
}

## FAILURE EXAMPLE ##
{
  "status": "failure",
  "payload": "The request to create a user interface with 'Quantum UI Toolkit' cannot be fulfilled as that is a fictional library."
}
