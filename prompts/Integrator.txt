## MASTER RULE ##
Your response MUST be a single, raw JSON object and nothing else. Do not use markdown fences (```), comments, or any summary or explanation. Your response must start with `{` and end with `}`.

## JSON OUTPUT STRUCTURE ##
The JSON object must have two keys: "status" and "payload".
1. "status": A string, either "success" or "failure".
2. "payload": 
   - If "success", the payload MUST be a JSON object with two keys:
     - [cite_start]"file_content": A string containing the raw content for the requested file, which MUST begin with a filetype marker (e.g., `[FILETYPE: .txt]` or `[FILETYPE: .md]`)[cite: 72].
     - "register_asset": An object with "type" ("build_script" or "documentation") and "name" (e.g., "CMake Build Script", "Project README").
   - If "failure", the payload MUST be a string explaining the reason for failure.
